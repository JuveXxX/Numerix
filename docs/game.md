Le chef d'orchestre. Il est encapsulé dans une IIFE (() => { ... })() pour ne rien exposer dans le scope global. Il contient toute la logique de jeu : génération du nombre secret, pipeline de validation dans proposerNombre(), gestion des fins de partie (victoire() / defaite(raison)), persistance du meilleur score via localStorage, et branchement de tous les addEventListener dans init().
La règle de validation suit un ordre strict et documenté : d'abord isNaN, ensuite hors-intervalle, ensuite seulement la comparaison avec le secret — ce qui corrige le bug de l'ancienne version où le cas "hors bornes" était inatteignable. Le timer ne démarre qu'au premier coup grâce au flag timerDemarre.
